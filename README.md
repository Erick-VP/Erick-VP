### How I think about engineering

* **Write code for humans, not just machines.**
    Maintainability is a core feature. I heavily prioritize clean code, solid documentation, and testing. A clever but unreadable solution is a technical debt.
* **Understand the "Why" before the "How".**
    Before jumping into the IDE, I focus on understanding the business requirement. Choosing between microservices or a monolith, SQL or NoSQL, depends entirely on the scale and context of the problem being solved.
* **Ownership from design to deployment.**
    I believe a developer's job doesn't end when the PR is merged. Monitoring logs, tracking performance, and ensuring the user experience doesn't break in production are just as critical as writing the logic.
